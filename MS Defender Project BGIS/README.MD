# Microsoft Defender for Endpoint Deployment – BGIS

## Overview

During my internship at BGIS, I supported the enterprise-wide deployment of Microsoft Defender for Endpoint across approximately 7,000+ corporate endpoints as part of a large-scale security modernization initiative.

The objective of this project was to improve endpoint protection, increase visibility across corporate devices, and ensure consistent security coverage across the organization.

---

## Problem Statement

A portion of corporate-managed devices did not have Microsoft Defender for Endpoint fully installed or properly configured. This created potential gaps in endpoint visibility and security monitoring.

The goal of the initiative was to:

* Identify affected users and devices
* Support onboarding and configuration
* Ensure consistent endpoint protection across all devices

---

## Security Risks Addressed

Devices without proper endpoint protection were exposed to risks such as:

* Malware and malicious applications
* Phishing and social engineering attacks
* Unsafe network connections
* Limited security monitoring and detection visibility

---

## My Role

As a Security Analyst Intern, I supported the deployment and onboarding process by working closely with the security and infrastructure teams.

My responsibilities included:

* Reviewing device inventories and identifying non-compliant endpoints
* Supporting user onboarding and configuration processes
* Assisting users with installation and setup issues
* Creating user communication materials and awareness messages
* Developing instructional guides and walkthrough videos
* Tracking deployment progress and user adoption
* Documenting issues and recurring challenges during rollout

---

## Deployment Approach

### Phase 1 – Planning & Assessment

* Reviewed deployment scope and requirements
* Analyzed device and user inventory
* Identified endpoints requiring onboarding
* Reviewed configuration and enrollment procedures

---

### Phase 2 – User Communication & Awareness

* Created and distributed user-facing communication emails
* Explained the purpose and benefits of Microsoft Defender deployment
* Provided step-by-step onboarding instructions
* Shared supporting documentation and instructional videos

---

### Phase 3 – User Support & Troubleshooting

Provided direct support to users experiencing issues during onboarding, including:

* Microsoft Defender sign-in issues
* Microsoft Authenticator configuration problems
* Device enrollment and permission errors
* Network-related installation issues
* Devices not enrolled in organization management systems

---

### Phase 4 – Tracking & Validation

* Monitored deployment progress across endpoints
* Tracked successful installations and onboarding completion
* Followed up with users requiring additional support
* Validated security configuration consistency across devices
